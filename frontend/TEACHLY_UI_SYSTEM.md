# TEACHLY — UI SYSTEM
Version 1.0
Mobile-First Only

---

## 🎯 Identidad

Teachly es un diario personal de aprendizaje.
Debe sentirse íntimo, cálido y reflexivo.

No es:
- App tech
- App corporativa
- App vibrante
- App tipo Duolingo

Es:
- Un espacio personal
- Un cuaderno moderno
- Una experiencia tranquila de lectura

---

# 📱 PRINCIPIO FUNDAMENTAL

DISEÑAR SOLO PARA MOBILE.

- Ancho máximo: 420px
- Centrado en pantalla
- Sin layouts desktop
- Sin sidebars
- Sin grids complejas

Todo debe funcionar en vertical.

---

# 🎨 PALETA OFICIAL

## Fondo principal
--bg: #F4F1EA   (crema tipo papel)

## Superficie (cards)
--surface: #FFFFFF

## Color estructural
--muted: #969B92   (gris verdoso suave)

## Texto principal
--text: #2F2F2F

## Texto secundario
--text-secondary: #6B7280

## Accent emocional
--accent: #BFA56A  (dorado apagado)

## Bordes suaves
--border: rgba(47, 47, 47, 0.12)

---

# 🔠 TIPOGRAFÍA

Fuente recomendada:
- Inter
- Manrope

Reglas:
- Nunca usar negro puro (#000)
- Títulos con font-weight 600
- Cuerpo con font-weight 400
- Tracking ligero en títulos
- Mucho espacio vertical

---

# 🧱 ESTRUCTURA GLOBAL

## Layout base

- Padding horizontal: 20px mínimo
- Padding vertical: 24px mínimo
- Border radius general: 16px
- Cards principales: 24px padding interno
- Espaciado entre bloques: 20–28px

Nunca elementos pegados.

---

# 🔘 BOTONES

## Botón primario

Fondo: --muted
Texto: blanco
Altura: 48px
Border radius: 16px
Sin sombras fuertes
Transición suave

## Botón secundario

Fondo: transparente
Texto: --accent
Subrayado opcional

Nunca usar azul estándar.

---

# 🧾 INPUTS

Altura: 48px
Border radius: 16px
Fondo: crema (--bg)
Borde: 1px sólido (--border)
Focus: halo suave con --accent 20% opacity

Sin outlines azules.

---

# 📖 PANTALLA "TEMA DEL DÍA"

## Estructura

Header:
- Logo minimal
- Indicador "Hoy"
- Racha discreta con color --accent

Card principal:
- Tag categoría
- Título grande
- Fecha
- Línea dorada fina bajo título
- Texto narrativo
- Sección "Puntos Clave"
- Botón "Marcar como leído"

## Reglas

- Mucho aire
- Texto no muy ancho
- Interlineado amplio
- Nada comprimido

---

# 🔥 RACHA

Color: --accent
Estilo:
- Ícono pequeño
- Número claro
- No exagerado

Debe sentirse logro personal, no gamificación infantil.

---

# 💬 CHAT

Ubicación:
- Fijo en parte inferior

Estilo:
- Fondo ligeramente distinto al bg
- Input redondeado
- Contador de preguntas restantes discreto
- Mensajes con burbujas suaves

No estilo WhatsApp.
Más estilo notas conversacionales.

---

# 📆 CALENDARIO

- Minimal
- Días leídos marcados con punto dorado
- Día actual resaltado con borde muted
- Sin colores vibrantes

---

# 🌫 SOMBRAS

Solo sombras suaves.
Nunca sombras duras o tipo material design.

---

# ✨ MICROINTERACCIONES

- Transiciones 150–200ms
- Fade suaves
- Botones con leve opacity change
- Nada rebote, nada bounce

---

# 🚫 PROHIBIDO

- Azul brillante
- Sombras pesadas
- Bordes sharp
- Gradientes fuertes
- Estética tech
- Iconografía pesada

---

# 🧠 SENSACIÓN FINAL

La app debe sentirse como:

Un diario personal de crecimiento intelectual.

Si un diseño se siente:
- Startup tech
- SaaS dashboard
- Aplicación educativa infantil

Está mal.

Debe sentirse:
Calma.
Espacio.
Reflexión.
Intimidad.
